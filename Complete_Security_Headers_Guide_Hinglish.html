Complete Security Headers Guide (Hinglish Version)

Ye document har security header ko simple Hinglish me explain karta hai — ye kya karta hai, developers kyu use karte hain, agar missing ho to kya risk hota hai, aur bug hunting mindset se iska kya matlab hai.

Content-Security-Policy

Ye Kya Karta Hai:

Browser ko batata hai kaunse scripts, styles aur resources load hone allowed hain. Mainly XSS attacks rokne ke liye.

Developers Kyu Use Karte Hain:

Developers in headers ko use karte hain taaki browser ko security instructions mile aur attack surface kam ho. Ye defense-in-depth strategy ka part hote hain.

Agar Missing Ho To:

Misconfiguration ya missing header ki wajah se XSS, clickjacking, data leakage, session hijacking ya information disclosure ka risk badh sakta hai — depend karta hai application context pe.

Bug Hunting Mindset:

Bug hunters check karte hain ki header properly configured hai ya nahi. Kabhi-kabhi misconfigured CORS, weak cookie flags, ya missing CSP real-world impact create kar sakte hain.
